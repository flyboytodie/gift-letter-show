# 项目开发文档：XXX的礼物展示及书信展示（情侣款·无服务器极简版）

## 一、项目概述
| 项目属性 | 具体内容 |
|----------|----------|
| 项目名称 | XXX的礼物展示及书信展示（情侣款） |
| 核心目标 | A方（男友）纯前端编辑礼物/书信/音乐内容，生成可保存的静态Web包/链接，B方（女友）打开后可保存整页，无服务器、极简高效 |
| 目标用户 | 一对一情侣（A方编辑，B方查看/保存） |
| 项目类型 | 纯前端静态Web应用（无后端/数据库/服务器依赖） |
| 核心特点 | 零服务器成本、编辑流程极简、展示页可保存、适配情侣温馨场景 |
| 核心约束 | 无需安全校验、无需时效性、仅提醒B方保存内容 |

## 二、核心需求规格

### 2.1 功能需求

#### （1）A方编辑页（核心，纯前端）
| 功能模块 | 具体要求 |
|----------|----------|
| 基础设置 | ✅ 自定义网站标题（无字符/字数限制，支持emoji）<br>✅ 一键切换页面主色调（温馨粉/简约白/治愈蓝，默认温馨粉） |
| 礼物管理 | ✅ 简单网格布局编辑：支持添加/删除礼物项<br>✅ 编辑流程：上传图片/视频 → 实时预览 → 填写备注（支持换行/emoji）<br>✅ 素材格式：仅支持图片和视频格式<br>✅ 卡片尺寸：统一尺寸，具体宽高可在页面上后期调整<br>✅ 图片预览：点击放大后为弹窗形式，包含导航控件<br>✅ 视频封面：自动生成默认封面图，支持从视频中截取设置<br>✅ 排序：拖拽调整礼物展示顺序（无需保存，即时生效）<br>✅ 素材操作：支持替换已上传的礼物素材（保留备注） |
| 书信编辑 | ✅ 生日祝福信编辑：标题+富文本正文<br>✅ 富文本格式：支持加粗、换行、字体颜色、斜体、下划线、列表、段落对齐等<br>✅ 字数限制：无字数限制<br>✅ 字体选择：手写体字体（考虑跨平台兼容性）<br>✅ 实时预览书信展示效果 |
| 背景音乐 | ✅ 上传MP3音频文件（仅支持MP3格式） → 实时预览播放<br>✅ 支持替换音频文件<br>✅ 播放控制：包含播放/暂停、音量调节控件 |
| 生成/保存 | ✅ 编辑完成后，点击「生成展示包」按钮，自动打包所有素材+展示页为ZIP压缩包<br>✅ ZIP命名规则：以A方编辑的网站标题作为压缩包名称<br>✅ 生成「临时预览链接」：采用简单免费方案实现，有效期为1天<br>✅ 自动保存：编辑过程中不会自动保存<br>✅ 内容保留：关闭浏览器后重新打开，编辑内容不会保留<br>✅ 提示语：「编辑完成！请将ZIP包发送给对方，并提醒TA保存展示页」 |

#### （2）B方展示页（核心，纯前端）
| 功能模块 | 具体要求 |
|----------|----------|
| 内容展示 | ✅ 礼物：简单网格布局展示（仅考虑PC端3列），图片点击放大、视频点击播放（带基础控制条），备注文字展示在素材下方<br>✅ 书信：温馨信纸样式展示（浅米色背景+手写字体），富文本格式还原<br>✅ 背景音乐：页面打开自动播放声音，有明显提示引导用户关闭声音，循环播放<br>✅ 播放控制：包含播放/暂停、音量调节控件<br>✅ 移动设备策略：针对移动浏览器自动播放策略进行优化适配 |
| 保存功能 | ✅ 页面底部显眼提示：「✨ 这份惊喜只为你～请点击浏览器「另存为」保存整页哦 ✨」<br>✅ 适配浏览器“另存为”功能：仅支持Edge浏览器，可保存整页（含素材）到本地 |
| 交互体验 | ✅ 图片懒加载、视频按需加载（减少加载压力）<br>✅ 页面加载：包含加载状态提示<br>✅ 无编辑入口、无多余功能，纯展示+保存提示 |

### 2.2 非功能需求
| 需求类型 | 具体要求 |
|----------|----------|
| 兼容性 | ✅ 编辑页：仅需兼容PC端Chrome/Edge浏览器（A方操作）<br>✅ 展示页：兼容Chrome/Edge/微信内置浏览器（B方查看）、PC/移动端<br>✅ 保存功能：适配主流浏览器“另存为”（Chrome/Edge/微信浏览器） |
| 性能 | ✅ 编辑页：单页面支持上传≤20个礼物素材（图片≤5MB/张、视频≤100MB/个、音频≤20MB/个），操作无明显卡顿<br>✅ 展示页：本地打开加载≤3秒，在线预览加载≤5秒（4G网络） |
| 视觉风格 | ✅ 整体温馨简约风：圆角卡片、柔和阴影、无复杂装饰<br>✅ 礼物卡片：统一尺寸、备注文字半透明背景（提升可读性）<br>✅ 书信样式：浅米色信纸纹理、手写体字体（如“华文行楷”） |

## 三、技术方案（无服务器·极简）

### 3.1 技术栈选择（零门槛·易开发）
| 技术类型 | 选型 | 选择理由 |
|----------|------|----------|
| 核心技术 | HTML5 + CSS3 + JavaScript（原生） | 零学习成本、易修改、无依赖，纯前端实现所有功能 |
| 构建工具 | Parcel | 零配置，开箱即用，支持自动打包和热更新，适合小型项目快速开发 |
| 辅助库 | 1. Quill（富文本编辑插件）<br>2. SortableJS（拖拽排序插件）<br>3. JSZip（前端打包ZIP插件）<br>4. Cropper.js（图片处理/视频封面截取）<br>5. Howler.js（音频播放控制） | 均为轻量级纯前端插件，无需服务器，实现核心编辑功能，API友好易于集成 |
| 布局方案 | Flex + Grid布局 | 快速实现响应式网格，适配PC/移动端 |

### 3.2 项目目录结构（极简·易操作）
gift-letter-show/ # 项目根目录（A 方本地解压即可使用）
├── package.json # 项目配置文件（包含Parcel构建工具配置）
├── edit.html # A 方编辑页（核心操作页）
├── show.html # B 方展示页（编辑完成后打包到 ZIP）
├── css/ # 样式文件夹
│ ├── edit.css # 编辑页样式
│ └── show.css # 展示页样式
├── js/ # 脚本文件夹
│ ├── lib/ # 依赖插件（已内置，无需额外下载）
│ │ ├── sortable.min.js # 拖拽排序
│ │ ├── quill.min.js # 富文本编辑
│ │ ├── jszip.min.js # 前端打包
│ │ ├── cropper.min.js # 图片处理/视频封面截取
│ │ └── howler.min.js # 音频播放控制
│ ├── edit.js # 编辑页核心逻辑（上传 / 预览 / 排序 / 打包）
│ └── show.js # 展示页核心逻辑（加载 / 播放 / 保存提示）
├── assets/ # 素材临时文件夹（编辑时自动生成）
│ ├── images/ # 礼物图片
│ ├── videos/ # 礼物视频
│ └── audio/ # 背景音乐
└── dist/ # 构建输出目录（生成ZIP包的来源目录）
